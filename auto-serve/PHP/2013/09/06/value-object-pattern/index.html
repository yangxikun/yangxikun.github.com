值对象模式

06 September 2013

学习自《Guide to PHP Design Patterns》

什么是值对象模式？

像PHP的整型那样运作：如果你把同一个对象资源赋值给两个不同的变量，然后改变其中的一个变量，另一个变量仍然不受影响。事实上，这就是Value Object模式的目标所在。(这个有点像PHP变量的引用计数，写时拷贝)

来看一个有bug的程序例子

  1 <?php
  2 /**
  3  * 货币类
  4  */
  5 
  6 class BadDollar {
  7 
  8   /**
  9    * 货币数量
 10    *
 11    * @var Float
 12    */
 13   protected $amount;
 14 
 15   /**
 16    * 构造函数
 17    *
 18    * @return Null
 19    */
 20   public function __construct( $amount = 0 ) {
 21     $this->amount = (float)$amount;
 22   }
 23 
 24   /**
 25    * 返回$amount的值 
 26    *
 27    * @return Float
 28    */
 29   public function getAmount() {
 30     return $this->amount;
 31   }
 32 
 33   /**
 34    * 将$amount的值增加
 35    *
 36    * @return Null
 37    */
 38   public function add( $dollar ) {
 39     $this->amount += $dollar->getAmount();
 40   }
 41 
 42 }
 43 
 44 /**
 45  * 工作类
 46  */
 47 class Work {
 48 
 49   /**
 50    * 工资
 51    *
 52    * @var Float
 53    */
 54   protected $salary;
 55 
 56   /**
 57    * 构造函数
 58    *
 59    * @return Null
 60    */
 61   public function __construct() {
 62     $this->salary = new BadDollar( 200 );
 63   }
 64 
 65   /**
 66    * 支付薪水
 67    *
 68    * @return class BadDollar
 69    */
 70   public function payDay() {
 71     return $this->salary;
 72   }
 73 
 74 }
 75 
 76 /**
 77  * 职工类
 78  */
 79 class Person {
 80 
 81   /**
 82    * 钱包
 83    *
 84    * @var class BadDollar
 85    */
 86   public $wallet;
 87 }
 88 
 89 /**
 90  * 测试类
 91  */
 92 class StackTest extends PHPUnit_Framework_TestCase{
 93   /**
 94    * 测试BadDollar类的正确性
 95    */
 96     public function testBadDollarWorking(){
 97         $job = new Work;
 98         $p1 = new Person;
 99         $p2 = new Person;
100         $p1->wallet = $job->payDay();
101         $this->assertEquals( 200, $p1->wallet->getAmount() );
102         $p2->wallet = $job->payDay();
103         $this->assertEquals( 200, $p2->wallet->getAmount() );
104         $p1->wallet->add($job->payDay());
105         $this->assertEquals( 400, $p1->wallet->getAmount() );
106         //this is bad — actually 400
107         $this->assertEquals( 200, $p2->wallet->getAmount() );
108         //this is really bad — actually 400
109         $this->assertEquals( 200, $job->payDay()->getAmount() );
110     }
111 }
112 ?>

从上面的代码和测试中可以看出,$p1和$p2拥有相同的BadDollar实例.为什么会这样呢?因为PHP5中,对象实例赋值给变量是按引用赋值的,且修改了对象实例的内容时,并不会像普通类型变量一样发生”写时复制,”值对象模式就是为了实现对象的”写时复制”.

解决方案

修改BadDollar类的add方法,如下

1 <?php
2 public function add( $dollar ) {
3     return new BadDollar( $this->amount + $dollar->getAmount() );
4 }
5 ?>

修改测试用例中的$p1->wallet->add($job->payDay());为$p1->wallet = $p1->wallet->add($job->payDay());.
