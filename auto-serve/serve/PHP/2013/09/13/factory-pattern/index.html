工厂模式

13 September 2013

学习自《Guide to PHP Design Patterns》

在面向对象编程中,我们实例化一个类最普遍使用的就是new操作符.但在某些情况下,new操作符也许不太好用.

例如,当你需要对一个变量进行赋值一个实例化对象时,例如有productA和productB两个类,变量需要根据条件赋值相应的类实例,那么也许代码会是这样子:

1 <?php
2 if( condition ){
3     $var = new productA();
4 }else{
5     $var = new productB();
6 }
7 ?>

如果使用了工厂模式的话,代码应该如下这样子:

1 <?php
2 $var = Product->create( $condition );
3 ?.

显然,代码简洁了许多,还有其他一些好处,比如现在productA不需要了,那么在代码中,你是否要得删除所有new productA的语句?但如果在工厂模式下,你只需要修改工厂的create方法.

一般会在以下几种情况考虑使用工厂方法:

1.当一个类的初始化需要经过许多复杂的计算或依赖于其他类时;

2.当需要动态地为变量实例化某个类的子类时;

工厂模式所带来的好处:增强系统的可扩展性和编码时尽量少的修改量.

简单的工厂示例:

 1 <?php
 2 /**
 3  * 产品基类
 4  * 
 5  * @abstract
 6  */
 7 abstract class Product {
 8 
 9   /**
10    * 获取产品的名字
11    * 
12    * @abstract
13    */
14   abstract function getName();
15 }
16 
17 /**
18  * A产品
19  */
20 class ProductA extends Product {
21 
22   /**
23    * 获取产品的名字
24    * 
25    * @return string
26    */
27   public function getName() {
28     echo 'Product A';
29   }
30 
31 }
32 
33 /**
34  * B产品
35  */
36 class ProductB extends Product {
37 
38   /**
39    * 获取产品的名字
40    * 
41    * @return string
42    */
43   public function getName() {
44     echo 'Product B';
45   }
46 
47 }
48 
49 /**
50  * 工厂类,用于生产产品
51  */
52 class ProductFactory {
53 
54   /**
55    * 创建product子类
56    * 
57    * @param mixed $condition 具体条件
58    * @return Product
59    */
60   public function create( $condition ) {
61     if ( $condition ) {
62       return new ProductA();
63     }else{
64       return new ProductB();
65     }
66   }
67   
68 }
69 ?>

复杂的工厂模式可能会定义接口类工厂或抽象类工厂,以及复杂的创建条件.
