注册表模式

29 September 2013

学习自《Guide to PHP Design Patterns》

注册表模式

注册表模式用于访问全局可重用的对象,可以容纳多个类的多个实例.这些类的实例一般都是经常被调用到或当做参数传递的.

基本的注册表模式用于保持全局存储,需要以下4种实现方法:

  1. Register::add()添加一个对象到注册表中,可以指定一个名称(一个类的多个实例)或者使用默认的类名(类似单例)
  2. Register::get()从注册表中检索一个对象
  3. Register::contains()在注册表中检查一个对象是否存在
  4. Register::remove()通过对象名在注册表中删除一个对象

实例代码:

 1 <?php 
 2 /**
 3  * 注册表类
 4  */
 5 class Register {
 6 
 7     /**
 8      * @static
 9      * @var array 存储实例
10      */
11     static private $_store = array();
12 
13     /**
14      * 添加一个对象到注册表中,可以指定一个名称(一个类的多个实例)或者使用默认的类名(类似单例)
15      * 
16      * @param mixed $object 被存储的对象
17      * @param string $name 用于检索对象的名字
18      * @return mixed 如果重写了实例,那么之前的实例会被返回
19      */
20     static public function add($object, $name = null) {
21         $name   = $name !== null ? : get_class($object);
22         $name   = strtolower($name);
23         $return = null;
24         if (isset(self::$_store[$name])) {
25             $return = self::$_store[$name];
26         }
27         self::$_store[$name] = $object;
28         return $return;
29     }
30 
31     /**
32      * 从注册表中检索一个对象
33      * 
34      * @param string $name 对象名字,{@see self::add()}
35      * @return mixed
36      * @throws Exception
37      */
38     static public function get($name){
39         if (!self::contains($name)) {
40             throw new Exception('对象没有存储在注册表中');
41         }
42         return self::$_store[$name];
43     }
44 
45     /**
46      * 在注册表中检查一个对象是否存在
47      * 
48      * @param string $name 对象名称,{@see self::add()}
49      * @return bool
50      */
51     static public function contains($name)
52     {
53         if (isset(self::$_store[$name])) {
54             return true;
55         }
56         return false;
57     }
58 
59     /**
60      * 通过对象名在注册表中删除一个对象
61      * 
62      * @param string $name 对象名称,{@see self::add()}
63      * @return void
64      */
65     static public function remove($name)
66     {
67         if (self::contains($name)) {
68             unset(self::$_store[$name]);
69         }
70     }
71 }
72 ?>
